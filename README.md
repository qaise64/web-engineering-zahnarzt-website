# Web Engineering Projekt: Zahnarztpraxis Website

Dieses Repository enthaelt ein vollstaendiges Web-Engineering-Projekt, das den gesamten Prozess von der Anforderungserfassung bis zur Implementierung abdeckt.

Das Projekt wurde im Rahmen der Module Web Engineering Praktikum 1 und Praktikum 2 an der Hochschule Niederrhein entwickelt.

**WICHTIG:** Die Live-Demo der implementierten Website finden Sie hier:
[**Live-Demo: Zahnarzt-Webseite**](https://zahnarzt-2e6381.pages.ide3.de/)

## Projekt-Phasen (Von Anforderung bis Code)

Das Projekt ist in drei Phasen unterteilt, die den Software-Lebenszyklus abbilden:

### 1. Konzeption & Anforderungserfassung (Praktikum 1)

* **Ergebnis:** Erstellung des **Lastenhefts** (definiert die Anforderungen des Kunden: *Was* soll das System koennen?).
* **Zusaetzlich:** Erstellung eines **Mockup-Designs** zur Visualisierung der Anforderungen. *Dies war keine offizielle Anforderung des Praktikums, sondern wurde zur besseren Planung des Projekts proaktiv erstellt.*

### 2. Spezifikation (Praktikum 2 - Teil 1)

* **Ergebnis:** Erstellung des **Pflichtenhefts** (definiert die technische Umsetzung: *Wie* soll das System gebaut werden?).

### 3. Implementierung (Praktikum 2 - Teil 2)

* **Ergebnis:** Umsetzung des Pflichtenhefts in funktionalen Code.

## Technologien der Implementierung

* **HTML**
* **CSS / JavaScript**

**Hinweis:** Dieses Projekt ist ein akademisches Beispiel fuer den Software-Lebenszyklus.
